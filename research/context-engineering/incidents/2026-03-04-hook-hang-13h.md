# Incidente: Hook auto-learnings colgado 13 horas

**Fecha:** 2026-03-04 | **Proyecto:** Context Engineering (hooks ecosystem)
**Tipo:** perdida_de_estado
**Severidad:** critico

## Descripcion
El hook `auto-learnings.ps1` se colgó durante ~13 horas tras el cierre de una sesión de Claude Code. El proceso `claude.exe` hijo quedó como huérfano consumiendo recursos.

## Contexto perdido
No se perdió contexto directamente, pero el proceso colgado consumió recursos y bloqueó el normal funcionamiento del sistema.

## Impacto
- Proceso `claude.exe` huérfano corriendo 13 horas
- Recursos del sistema consumidos innecesariamente

## Causa raiz (4 causas compuestas)

### CR-1: Timeout en settings.json en unidad incorrecta (CRITICA)
El campo `timeout` en hooks de Claude Code es en **segundos**, no milisegundos. `timeout: 120000` = 33.3 horas, no 2 minutos. Todos los hooks tenían este error.

### CR-2: `claude --print` invocado sin timeout interno
La variable `$CLAUDE_TIMEOUT = 60` estaba declarada pero nunca se usaba. La llamada `$prompt | & claude --print` era sincronica sin limite de tiempo.

### CR-3: Windows no mata procesos hijos automáticamente
Al matar `pwsh.exe` (si el timeout hubiera funcionado), `claude.exe` hijo no se mata automáticamente en Windows. No hay equivalente a `PR_SET_PDEATHSIG` de Linux.

### CR-4: Pipe stdin sin EOF explícito
`$prompt | & claude --print` depende de PowerShell para señalizar EOF. En contexto de Stop hook, el manejo de pipes es no-determinístico.

## Mitigacion aplicada

### settings.json - Todos los timeouts corregidos
| Hook | Antes (interpretado) | Despues |
|------|---------------------|---------|
| dashboard-update | 5000s (83 min) | 5s |
| session-health-check | 10000s (2.7h) | 10s |
| auto-learnings | 120000s (33.3h) | 120s |
| precompact-persist | 120000s (33.3h) | 120s |

### auto-learnings.ps1 - Reescrito hang-proof
1. **Timeout interno:** Start-Job + Wait-Job con 45s inner + 55s outer
2. **Cleanup recursivo:** `Stop-ClaudeChildren` mata todo el árbol de procesos descendientes
3. **Stdin via temp file:** Evita ambigüedad de EOF en pipes
4. **Guard de re-entrancia:** `$env:CLAUDE_AUTO_LEARNINGS_RUNNING` previene loops infinitos

### precompact-persist.ps1 - Mismo fix aplicado
Mismas 4 protecciones con timeout de 90s.

## Prevencion
- Documentar en LEARNINGS que hook timeouts son en SEGUNDOS
- Aplicar el mismo patrón (Start-Job + temp file + cleanup + guard) a cualquier hook futuro que invoque `claude --print`
