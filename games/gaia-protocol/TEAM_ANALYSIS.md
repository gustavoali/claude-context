# Gaia Protocol - Equipo de Analisis

## Composicion

| Rol | Agente | Responsabilidad |
|-----|--------|-----------------|
| Arquitecto General | `software-architect` | Arquitectura del sistema completo (Unity + Core + Backend) |
| Arquitecto Backend | `software-architect` + perfil backend | Diseno de engines, modelo de dominio, API |
| Product Owner | `product-owner` | Backlog, user stories, priorizacion, epicas |
| Project Manager | `project-manager` | Planning, sprints, milestones, riesgos |
| Business Stakeholder | `business-stakeholder` | Validacion de scope, decisiones de negocio, GO/NO-GO |
| Database Expert | `database-expert` | Modelo de persistencia, schemas, optimizacion |

## Notas
- El arquitecto general define como se comparte el nucleo C# entre Unity y backend
- El PO gestiona el backlog basado en los milestones M0-M6 definidos en el seed
- El PM coordina los 7 sprints propuestos en el roadmap
- El business stakeholder valida scope del MVP y decisiones de corte
